# Code of Conduct

## Our Standards

This project is maintained as a practical engineering workspace. Contributors are expected to communicate clearly, respectfully, and constructively.

Examples of acceptable behavior:

- Using professional and respectful language.
- Giving clear technical feedback.
- Focusing on the code, documentation, and project goals.
- Reporting problems with enough detail to reproduce them.
- Respecting operational security and privacy.

Examples of unacceptable behavior:

- Harassment, insults, threats, or personal attacks.
- Publishing private, sensitive, or operational information.
- Deliberately disrupting the project.
- Submitting malicious code or unsafe changes.
- Ignoring security boundaries.

## Security and Sensitive Information

Do not publish secrets, credentials, tokens, private spreadsheet IDs, personal data, military-sensitive information, or operational details in issues, pull requests, commits, or logs.

If sensitive information is accidentally committed, remove it immediately and rotate the exposed secret.

## Enforcement

Project maintainers may remove comments, reject contributions, close issues, block users, or take other reasonable actions to protect the project and its users.
