# Архітектура зведення дня (коротке та детальне)

Цей проєкт працює в Google Apps Script (V8) і прив’язаний до Google Sheets. **Джерелом розрахунків є формули в таблиці**, а скрипт лише **зчитує готові значення** й формує текст повідомлення.

## Принцип

Формули в Google Sheets рахують → Apps Script читає готові значення → Apps Script формує повідомлення

## Місячні листи

Місячні листи мають назви `01`…`12`. На кожному:

- **рядок дат** (зазвичай рядок 1) з датами місяця;
- **матриця графіка** (коди по людях і днях);
- **нижній формульний блок** (після зони графіка) з підсумками по кожному дню.

Нижній блок містить рядки-показники (у таблиці можуть бути технічні ключі з `_`):

- `За_списком`
- `В_наявності`
- `У_відрядженні`
- `У_відпустці`
- `Гусачівка`
- `Drone_Camp`
- `ППД`
- `КП`
- `БР`

Показник `За_штатом` може існувати в таблиці, але **в коротке зведення не виводиться**.

## Коротке (звичайне) зведення дня

**Що робить:**

- визначає місячний лист за датою (наприклад `14.06.2026` → лист `06`);
- знаходить колонку цієї дати в рядку дат (не жорстко по номеру колонки);
- знаходить нижній формульний блок зведення **динамічно** (не прив’язуючись до конкретної колонки B/C);
- зчитує готові значення перетином:
  - **рядок показника** × **колонка дати**;
- форматує текст у фіксованому порядку показників.

**Чого НЕ робить:**

- не рахує заново через PERSONNEL / DICT_SUM;
- не дублює логіку формул таблиці.

## Детальне зведення дня

Детальне зведення — окрема логіка, яка:

- читає місячний лист + конкретну колонку дня;
- проходить людей/коди графіка;
- використовує PERSONNEL та DICT_SUM (групування/порядок/лейбли);
- формує розшифровку: **хто де знаходиться** і по яких кодах.

Детальне зведення не підміняє коротке і не змішується з ним.

## Модулі та відповідальність

### `Report_SummaryData.gs` (читання)

- константи: `SIMPLE_DAILY_SUMMARY_ORDER`, `SIMPLE_DAILY_SUMMARY_LABELS`
- нормалізація: `normalizeSummaryKey_()` (пробіли → `_`, злиття `_+`)
- парсер: `parseSummaryNumber_()` (`"29 осіб"` → `29`, порожнє → `0`)
- пошук блоку: `findSummaryBlockLocation_()` (динамічно)
- пошук колонки дати: `findDateColumnInMonthSheet_()`
- читання значень: `readDailySummaryFromFormulaBlockForSheet_()` / `readDailySummaryFromFormulaBlock_()`

### `Report_DailySimple.gs` (форматування)

- `formatSimpleDailySummary_()`
- `buildSimpleDailySummaryFromFormulaBlock_()`

### `Report_DailyDetailed.gs` (детальне)

- `collectPeopleDetailed_()`
- `formatDetailedSummary_()`
- допоміжні функції для детального розпису

### `Summaries.gs` (legacy-сумісність)

- `buildDaySummaryForColumn_()` лишається публічною точкою входу, але **делегує** в коротке зведення з формульного блоку.

## UI / запуск

- API: `apiBuildDaySummary`, `apiBuildDetailedSummary`
- Меню: окреме меню `Звіти` з пунктами показу короткого та детального зведення.
