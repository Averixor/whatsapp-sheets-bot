# WASB — інтегрований пакет

## Що об'єднано

Проєкт Google Apps Script тепер працює з основною книгою взводу та журналом тимчасово виданого майна як з єдиною системою.

Під час одноразового налаштування система:

- створює резервну копію старого журналу майна;
- мігрує п'ятиколонковий аркуш `Property_issued_for_temporary_u` з основної книги;
- за потреби імпортує аркуш `Облік майна`, якщо його скопійовано до тієї самої Google-таблиці;
- створює довідники `PROPERTY_CATALOG` і `PROPERTY_KITS` без видалення власних записів;
- створює пов'язане комплектне майно окремими службовими рядками;
- зберігає фактичний стан повернення окремих комплектуючих під час міграції;
- створює `PROPERTY_SUMMARY` та `PROPERTY_OUTSTANDING`;
- синхронізує позивні з `PERSONNEL`;
- показує неповернуте майно в картках особового складу.

Детальніше про модель даних і CI: [`temporary-property-register.md`](./temporary-property-register.md).

## Розгортання

Потрібен Node.js 24 або новіший.

```bash
npm ci
npm run ci
npx clasp push
```

Після завантаження коду відкрийте редактор Apps Script прив'язаної Google-таблиці та один раз запустіть:

```javascript
apiSetupTemporaryPropertyRegister()
```

Потім перезавантажте Google-таблицю.

## Важливо

Основна книга вже містить старий журнал `Property_issued_for_temporary_u`, тому окремо переносити дані з файла `oblik_maina_profesiinyi_dyzain(1).xlsx` не потрібно. Цей файл залишено в локальній теці [`reference-workbooks/`](../reference-workbooks/) як еталон дизайну та для звірки (не входить до CI; контракти вже в `contracts/`).

Не запускайте налаштування на копії без перевірки резервного аркуша, якщо в робочому журналі вже є нові записи після моменту створення завантажених файлів.
